小红书知识剪藏

长期以来我所面对的注意力问题使我产生了一定的“短篇幅知识”依赖，即通过刷小红书或微信公众号等来获取“知识”，虽然我自己也意识到很大程度上这些碎片内容除了当当谈资毫无用处，但它们多少给我sense of fulfillment。

因此我希望多少能用最少的努力让这些东西不是左眼进右眼出。

因此我希望设计一套流程，能让我尽可能无需过多的操作就获得归档好的信息以及整理好的知识。
为了实现这两个目标，我们目前的想法分为两部分，一是实现从手机上刷的内容到电脑表格，二是从表格里的链接到chatgpt总结。

手机 to 电脑

这一步主要用于把零散信息集中化，表格主要作为日后查询有用内容的集装箱以及知识的中转站。

使用苹果的快捷指令+飞书多维表格api，主要参考结合两者做的记账应用教程。

  • https://www.bilibili.com/video/BV1Ew411V75U/
  • https://mp.weixin.qq.com/s/6vm4atnzQQMT3kCcYB4RqQ

手机端流程为小红书点击分享链接，苹果启动快捷指令监听剪贴板上的内容，识别链接部分并用浏览器中的阅读器打开以获得标题，然后调用飞书接口（输入app id secret id等），以将标题、链接和备注信息同步到多维表格中。

表格to文档

这一步主要用于把信息转为知识，通过文档化让AI提取更多insight或可用内容。

首先涉及抓取完整小红书内容，由于这里基本不涉及批量搜集，对性能没有要求所以尽可能减少折腾用了[[Selenium]]，将当前需要一起整理的文档输出为 “标题-文字内容-图片/视频封面” 的结构，并储存到markdown文件中。

简单校对与检查后生成pdf并提交给ChatGPT，可根据具体情况提问，如简单总结或帮助列表整理商品信息等。

待定步骤// 生成的pdf回传到飞书以和表格建立关联？ChatGPT近期有了项目功能，是否要利用其特性？

微信公众号与博客更新

输出和输入同等重要，先完成再完美。

二者的区别在于博客更方便随时更新（用的很古早的[[hexo]]…) 所以博客内容可以相对整合，公众号主要会把不再更新的部分整体发上去。

随笔

一些相对随意的内容可以按体量决定单独发出来还是合并到两周左右发一次

格式

统一先整理到obsidian再上传。
标题后不空行，段落间空行。尽量只使用加粗和少量的斜体。文章标题为一级，后续二三四级跟上。

图片处理

微信公众号不支持链接图片的显示：尽可能减少图片使用+本地上传（直接右键复制本地图片再粘进编辑器）

博客主要使用图床，具体使用 [[obsidian]] + [[picgo]] + [[github]]。虽然之前Typora就配置过一次，logseq又配置过一次，没想到转到obsidian又要配置一次。 Github token又过期了都
